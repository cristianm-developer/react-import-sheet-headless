# 4. Convert — Construction Plan

## Objective

Add a **Convert** step immediately after the Parser. It receives the **raw parse result** (`RawSheet`), the **sheet layout** (provided when instantiating the main hook), and tries to produce a **layout‑aligned sheet**: same structure as `RawSheet` but with row cells keyed by the layout’s field names. When the file headers do not match the layout (wrong column names or order), Convert does **not** fail the flow; it exposes the **headers found**, **mismatch details** (expected vs found), and **APIs to reorder columns** and **rename (map) columns** so the consumer can fix the mapping and re-run conversion.

---

## Position in the pipeline

- **2. Setting** defines shared types and the provider; the main hook receives **sheetLayout** at instantiation.
- **3. Parser** produces **RawParseResult** → selected sheet is **RawSheet** (e.g. first sheet); `rawData` in state.
- **4. Convert** (this step) consumes **RawSheet** + **sheetLayout**; outputs either a **ConvertedSheet** (fit) or a **ConvertResult** with mismatch info and correction APIs.
- **5. Sanitizer** (next step) consumes the **output of Convert** (ConvertedSheet), not raw parser output.

Flow: **Parser → Convert → Sanitizer → Validator → Transform** (and Edit on result).

---

## Input and output

| Concept               | Source                        | Description                                                                                                                                                   |
| --------------------- | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Input**             | Parser + Setting              | **RawSheet** (from parser, e.g. `rawData`) and **sheetLayout** (from provider / main hook).                                                                   |
| **Output (fit)**      | ConvertedSheet                | Same shape as RawSheet but row cells are keyed by **layout field names**; each cell key is a `sheetLayout.fields[].name`. Ready for Sanitizer.                |
| **Output (mismatch)** | ConvertResult (needs mapping) | Headers found, list of mismatches (expected field, what was found or missing), and APIs to **reorder** and **rename** columns so the user can fix and re-run. |

---

## Layout and “expected” columns

From **2. Setting**, **sheetLayout** has **fields**: each field has at least `name` (and optionally `description`, `example`, validators, transformations, `valueType`, `inputType`, **`required`**). The **expected columns** for Convert are the **field names** from the layout (in the order defined by the layout, unless overridden by a column-order preference).

- **Per field:** **`required?: boolean`** — when **true** or omitted, the column must be present or mappable; when **false**, the column is optional (missing columns do not block conversion).
- **Perfect fit**: every **required** layout field has a matching header in `RawSheet.headers` (case-insensitive or normalized match), and the order can be inferred (by index or by explicit order). Optional fields may be missing; they appear in the converted sheet with **null** values.
- **Extra file columns**: Columns in the file that do not map to any layout field are **ignored** (not an error). Only layout fields appear in **ConvertedSheet**.
- **Mismatch**: at least one **required** layout field has no matching header, or the user’s file has different header names/order and the consumer wants to **reorder** or **map** (rename) file headers to layout fields. When any required column is missing, the result includes **`layoutError: true`** so the UI can show a total layout/format error (e.g. document has 3 columns but layout requires 5).

---

## Contract: types and result shapes

### ConvertedSheet (success)

Structure aligned with **RawSheet** but with cell keys coming from the layout:

- **headers**: same as **layout field names** in the chosen order (so downstream steps see a stable key set).
- **rows**: each row has `cells` where each cell’s `key` is a layout **field name** and `value` is taken from the row’s cell that was mapped from the file (by header name or by index after reorder/rename).
- All other RawSheet metadata (`name`, `filesize`, `documentHash`, `rowsCount`, `headersCount`, etc.) are preserved.

So **ConvertedSheet** is the type that Sanitizer receives: same as RawSheet in shape, but keys are layout field names and values come from the mapped columns.

### ConvertResult when not a perfect fit

When the file headers do not match the layout (or the user has not yet applied a column mapping), Convert returns a result that allows the UI to show problems and offer correction:

- **headersFound**: `string[]` — list of headers as they appear in the file (from `RawSheet.headers`).
- **mismatches**: list of **ColumnMismatch**:
  - **expected**: `string` — layout field name (the column the layout expects).
  - **found**: `string | null` — header from the file that was matched (if any); `null` if missing.
  - **required**: `boolean` — **true** when the layout field is required (default); **false** when the field has `required: false`.
  - **message**: optional human-readable message (e.g. “Column ‘Email’ not found in file”).
- **layoutError**: `boolean` (optional) — **true** when at least one **required** column is missing (insufficient columns for the layout); use in the UI to show a total layout/format error.
- **columnOrder**: `string[]` — current order of **layout field names** for display/export (default: layout order). Used when reordering columns.
- **headerToFieldMap**: `Record<string, string>` — map from **file header** → **layout field name**. Enables “rename”/“map this column to that field”.
- **reorderColumns(fieldNames: string[])**: function to set the **display/processing order** of layout columns (updates `columnOrder`).
- **renameColumn(fileHeader: string, layoutFieldName: string)**: function to map a file header to a layout field (updates `headerToFieldMap`). If `layoutFieldName` is already mapped by another header, behavior is defined (e.g. last-wins or replace).
- **applyMapping()**: function that, using current `headerToFieldMap` and `columnOrder`, re-runs the conversion and returns either a new **ConvertedSheet** (success) or the same **ConvertResult** shape with updated mismatches if still invalid.

So: the hook/main API exposes either a **ConvertedSheet** (ready for Sanitizer) or a **ConvertResult** (headers found, mismatches, reorder/rename and apply).

---

## Behaviour summary

1. **On run**: Input = **RawSheet** + **sheetLayout**.
2. **Matching**: For each layout field, try to find a header in `RawSheet.headers` (exact or normalized, e.g. trim + case-insensitive). If a prior **headerToFieldMap** exists (e.g. from previous user correction), use it first.
3. **If all fields matched** (and optional order is satisfied): output **ConvertedSheet**; pipeline can continue to Sanitizer.
4. **If any field is missing or ambiguous**: output **ConvertResult** with:
   - **headersFound** = `RawSheet.headers`,
   - **mismatches** = list of expected vs found,
   - **columnOrder** / **headerToFieldMap** from last run or defaults,
   - **reorderColumns**, **renameColumn**, **applyMapping** so the consumer can fix and retry.
5. **Re-run**: When the user reorders or renames, the consumer calls **reorderColumns** / **renameColumn** and then **applyMapping()**; the result is again either **ConvertedSheet** or **ConvertResult**.

---

## Where Convert runs (main thread vs worker)

- **Convert** is **lightweight** (no heavy parsing): it only aligns headers and builds new row cell arrays. It can run on the **main thread** to avoid extra Worker round-trips and to keep mapping state (reorder/rename) in the same place as the UI.
- If the sheet is very large, the implementation can still move the “build ConvertedSheet” step into a Worker and keep only the mismatch result and the correction APIs on the main thread; the plan does not require it for v1.

---

## Integration with Setting and main hook

- The **main hook** (e.g. `useImportSheet`) is instantiated with **sheetLayout**. So Convert always has access to the layout.
- **rawData** (RawSheet) comes from the Parser (via provider state). When the user triggers “convert” (e.g. after preview or after file load), the hook runs Convert with `rawData` + `sheetLayout`.
- **State to expose** (from provider or hook):
  - Either **convertedSheet: ConvertedSheet | null** (when fit) so Sanitizer can run,
  - Or **convertResult: ConvertResult** (when mismatch) so the UI can show headers found, errors (expected column), and call **reorderColumns** / **renameColumn** / **applyMapping**.
- After a successful **applyMapping()** or a successful initial conversion, the provider/hook can set **convertedSheet** and advance status so the pipeline continues to Sanitizer.

---

## API shape (summary)

- **convert(rawSheet: RawSheet, sheetLayout: SheetLayout, options?: ConvertOptions): ConvertSuccess | ConvertResult**
  - **ConvertSuccess**: `{ kind: 'success', sheet: ConvertedSheet }`
  - **ConvertResult** (mismatch): `{ kind: 'mismatch', headersFound, mismatches, columnOrder, headerToFieldMap, reorderColumns, renameColumn, applyMapping }`
- **reorderColumns(fieldNames: string[])**: sets the order of layout fields used for building the converted rows and for display.
- **renameColumn(fileHeader: string, layoutFieldName: string)**: maps a file header to a layout field; used in the next **applyMapping**.
- **applyMapping()**: re-runs conversion with current **headerToFieldMap** and **columnOrder**; returns **ConvertSuccess** or **ConvertResult**.

## Plan of tasks (suggested order)

### Phase A — Types and contract

1. **Types in `src/types` or `core/convert/types`**
   - **ConvertedSheet**: same structure as RawSheet; `headers` = layout field names (in chosen order); each row’s cells keyed by layout field name, values from file.
   - **ColumnMismatch**: `{ expected: string, found: string | null, message?: string }`.
   - **ConvertSuccess**: `{ kind: 'success', sheet: ConvertedSheet }`.
   - **ConvertResult** (mismatch): `{ kind: 'mismatch', headersFound: string[], mismatches: ColumnMismatch[], columnOrder: string[], headerToFieldMap: Record<string, string>, reorderColumns, renameColumn, applyMapping }`.
   - **ConvertOptions** (optional): e.g. case-sensitive matching, normalizer for headers.

2. Document in **2. Setting** / **Architecture** that the pipeline is Parser → **Convert** → Sanitizer → Validator and that Sanitizer receives **ConvertedSheet**.

### Phase B — Core conversion logic

3. **matchHeadersToLayout(rawSheet, sheetLayout, headerToFieldMap?)**
   - For each layout field, resolve the file header (from map or by name match). Return a mapping: layout field name → file header (or index), and list of mismatches.

4. **buildConvertedSheet(rawSheet, sheetLayout, columnOrder, headerToFieldMap)**
   - Build ConvertedSheet: for each row, build cells keyed by layout field names, values taken from the row’s cell for the mapped file header. Preserve RawSheet metadata.

5. **runConvert(rawSheet, sheetLayout, options?)**
   - Run matchHeadersToLayout; if no mismatches, return **ConvertSuccess** with buildConvertedSheet output.
   - If mismatches, return **ConvertResult** with headersFound, mismatches, default columnOrder (layout order), empty headerToFieldMap, and functions that close over state to implement **reorderColumns**, **renameColumn**, **applyMapping**.

### Phase C — Hook and provider integration

6. **Hook (e.g. useConvert or integrated in useImportSheet)**
   - Reads `rawData` (RawSheet) and `sheetLayout` from context; exposes convert result (ConvertedSheet or ConvertResult).
   - When **ConvertResult**: exposes **reorderColumns**, **renameColumn**, **applyMapping** so the consumer can fix mapping and re-run.
   - On success, updates provider (e.g. `convertedData` or equivalent) so Sanitizer can consume ConvertedSheet.

7. **Provider / state**
   - Add slot for **convertedSheet** (or **convertResult**) and status for “needs mapping” vs “converted”; document in Architecture.

### Phase D — Tests

8. **Unit tests**
   - **matchHeadersToLayout**: perfect match (all layout fields found); missing column (expected present, found null); case/normalization; use of headerToFieldMap.
   - **buildConvertedSheet**: correct keys (layout field names), values from correct file column; columnOrder respected.
   - **runConvert**: returns ConvertSuccess when headers match; returns ConvertResult with correct headersFound and mismatches when they do not; after renameColumn + applyMapping, returns ConvertSuccess when mapping is complete.
   - **reorderColumns** / **renameColumn** / **applyMapping**: reorder changes column order in output; rename maps file header to field and applyMapping produces ConvertedSheet when all fields mapped.
